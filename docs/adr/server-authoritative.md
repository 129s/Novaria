# Server-Authoritative 网络模型

- `status`: accepted
- `owner`: @novaria-core
- `last_verified_commit`: 30e0b63
- `updated`: 2026-02-06

## 背景

项目目标包含联机一致性与可扩展 Mod 共存。若客户端直接改写世界真值，会放大不同步与作弊风险。

## 决策

采用 `Server-Authoritative` 模型：

- 客户端发送命令，不直接提交世界真值。
- 权威状态由服务端（MVP 阶段为房主兼服）推进并复制。
- 断线重连后以权威状态为准恢复。

## 备选方案

- P2P 对等真值：实现简单，但一致性与安全性成本高。
- Client-Authoritative：延迟观感好，但作弊与错乱风险高。

## 后果与代价

- 优点：一致性边界清晰，便于联机与 Mod 扩展。
- 代价：需要额外处理命令确认、状态复制、重连恢复。

## 回退条件

无。若后续演进为专用服，该决策保持不变，仅替换承载形态。
