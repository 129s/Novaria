# Novaria 开工日清单（Day 1）

## 1. 当日目标

在 Day 1 结束时，项目必须具备“可编译、可运行、可扩展”的最小骨架：

- 编译系统可用。
- 程序入口可运行。
- 日志与配置可读。
- 目录结构冻结并可持续扩展。

## 2. 任务清单（按优先级）

## 2.1 P0：必须完成

- [ ] 建立仓库基础结构：`src/`、`assets/`、`mods/`、`docs/`、`tools/`、`tests/`。
- [ ] 初始化 CMake 根工程，支持 `Debug/Release`。
- [ ] 集成 SDL3（链接成功，窗口创建成功，事件循环可跑）。
- [ ] 创建 `main` 入口与最小 Game Loop（固定 Tick + 渲染帧）。
- [ ] 建立日志模块（至少支持 `info/warn/error`）。
- [ ] 建立配置加载入口（如 `config/game.toml`）。

## 2.2 P1：建议完成

- [ ] 引入基础断言与错误码规范。
- [ ] 建立最小单元测试入口（先空测试也可）。
- [ ] 建立 `mods/_example_core` 空模板目录。

## 3. 交付物定义

Day 1 结束后需提交：

1. 可编译运行的工程骨架。
2. 基础目录结构与模块占位文件。
3. 一份简短构建说明（命令级别）。
4. 一份运行截图或日志片段（证明主循环在跑）。

## 4. 验收标准（严格）

同时满足以下条件才算 Day 1 完成：

- 在干净环境中可完成一次完整构建。
- 程序启动后能打开窗口并稳定运行 60 秒不崩溃。
- 日志包含启动、主循环、退出三个关键事件。
- 关闭程序后无明显资源释放错误日志。

## 5. 禁止事项

- 禁止在 Day 1 写业务玩法逻辑。
- 禁止提前引入“看起来很酷但不必要”的系统。
- 禁止把框架问题拖到后续 Sprint。

## 6. Day 2 起步条件

仅当 Day 1 验收通过，才进入 Day 2：

- 开始世界数据结构与 Chunk 基础实现。
- 同步定义 `sim/world/net/script` 的首批接口契约。
