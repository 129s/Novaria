# Novaria MVP 定义

- `status`: authoritative
- `owner`: @novaria-core
- `last_verified_commit`: 1937135
- `updated`: 2026-02-06

## 1. 目标

Novaria 的 MVP 目标是验证三条闭环，而不是追求内容体量：

- 可玩闭环：采集 → 建造 → 战斗 → 击败小 Boss → 存档回读。
- 技术闭环：固定 Tick、稳定可观测、可持续迭代。
- 扩展闭环：联机与 Mod 在同一架构下可验证一致性。

## 2. 平台与技术

- 目标平台：Windows（MVP 阶段仅此平台）。
- 语言：C++20。
- 底层框架：SDL3。
- 脚本定位：LuaJIT（规划目标），当前工程仍以 `ScriptHostStub` 为主。

## 3. 完成定义（DoD）

1. 新建世界后可完成采集、建造、战斗、击败小 Boss，并可存档回读。
2. 1080p 场景下客户端 60 FPS，P95 帧时间 ≤ 16.6ms。
3. 4 人协作连续 30 分钟无致命崩溃，关键状态无持续错乱。
4. 第三方 Mod 可新增 `item/recipe/npc` 行为并通过一致性校验。

## 4. 范围边界

### 4.1 MVP 必做

- 世界：有限地图、Chunk 管理、基础地形分层。
- 玩法：挖掘/放置、工作台、基础合成、基础战斗、1 个小 Boss。
- 联机：Host-Authoritative 基线、会话管理、心跳与重连。
- 存档：版本字段、兼容加载、关键诊断快照。
- Mod v0：清单、依赖、内容定义（`item/recipe/npc`）、一致性指纹。

### 4.2 MVP 不做

- PVP、匹配大厅、跨平台联机。
- 大型剧情任务线与复杂世界事件系统。
- 严格脚本安全沙箱。
- 脚本改写底层渲染或网络协议。
